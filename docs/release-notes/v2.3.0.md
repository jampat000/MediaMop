# MediaMop v2.3.0

Date: 2026-05-13

This release replaces the Windows updater service with a native .NET tray app and Velopack, delivering automatic delta updates and a public documentation site.

## What Changed

- **Windows desktop app**: MediaMop now ships as a native .NET tray application. The tray icon provides Open MediaMop, Open Data Folder, and Quit. The old Python tray host is gone.
- **Automatic delta updates**: Velopack handles Windows updates in the background. Delta packages keep downloads small and rollback is automatic on failure. No admin rights required.
- **Settings → Upgrade**: The upgrade panel now shows the running version and latest release. Windows updates are applied automatically by the tray app — no manual trigger needed.
- **Public documentation site**: https://jampat000.github.io/MediaMop/ covers quickstart, deployment, architecture, and API reference.

## Fixes And Stability

- Removed the updater service process entirely, eliminating a class of background service failures.
- Backend no longer makes inbound HTTP calls to a local updater service; update status is a direct GitHub release check only.

## Upgrade Notes

- **Windows**: If you are upgrading from v2.2.x or earlier, uninstall the old MediaMop (via Add/Remove Programs) and run the new `MediaMop-*-win-Setup.exe` installer. The new installer is per-user (`%LocalAppData%\MediaMop`) and requires no admin rights. Runtime data under `C:\ProgramData\MediaMop` is preserved. The tray app will automatically clean up any legacy updater service on first launch.
- **Docker**: No changes to the container image or volume layout. Pull the new tag and restart.
- No database or configuration migration is required when upgrading.
