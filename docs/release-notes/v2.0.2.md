# MediaMop v2.0.2

Date: 2026-05-06

This release focuses on Windows upgrade reliability and packaging stability.

## What Changed

- Fixed stale app shell caching after upgrades by forcing browser revalidation of `index.html`.
- Improved Windows packaging reliability by invoking PyInstaller via `python -m PyInstaller` in the build script.
- Reduced unnecessary bundled runtime extras by switching backend dependency from `uvicorn[standard]` to `uvicorn`.

## Fixes And Stability

- `index.html` now returns explicit no-cache headers for `/`, `/index.html`, and SPA history refresh routes.
- Added UPX exclude patterns for Python extension and runtime DLL files to reduce AV false positives and runtime import issues on Windows.
- Confirmed Windows packaging still includes required `.pyd` and `.dll` binaries and launches normally.

## Upgrade Notes

- If you are upgrading from an older Windows install that predates the updater service, run `MediaMopSetup.exe` once as administrator.
- After that one-time bootstrap, future upgrades can be started from **Settings -> Upgrade**.
- No additional operator action is required for this release.
