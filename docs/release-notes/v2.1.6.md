# MediaMop v2.1.6

This patch release fixes the Windows desktop relaunch path after an in-app upgrade so MediaMop does not report a clean desktop upgrade while the tray app is still down.

## Highlights

- Fixed the Windows active-session tray relaunch path used after a silent in-app upgrade.
- Tightened Windows upgrade verification so an active desktop session must get the tray host back before the upgrade is treated as a clean success.
- Kept the verified backend-version completion gate, checksum validation, and trusted installer download path from earlier 2.1.x releases.
- Preserved upgrade diagnostics so tray relaunch failures remain debuggable without hiding the real reason.

## Operator notes

- On Windows desktop installs with an active user session, MediaMop no longer treats a backend-only restart as a fully successful desktop upgrade.
- This release specifically addresses the case where the backend comes back on the new version but the tray app does not relaunch.
