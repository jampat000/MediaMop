# MediaMop v2.3.1

Date: 2026-05-14

Patch release fixing the Windows installer upgrade experience and tray reliability.

## What Changed

- **Installer upgrade path**: Installing v2.3.x over a running v2.2.x instance now works cleanly. The install hook kills any running MediaMop processes before starting the new tray, so no manual process-killing is required.
- **Automatic startup registration**: The tray now registers itself in `HKCU\Run` on install and removes the entry on uninstall. Reboots work correctly without any manual setup.
- **Server auto-restart**: If the bundled server crashes, the tray automatically restarts it with exponential backoff (2s → 5s → 15s → 30s → 60s, up to 5 attempts) before surfacing an error to the user.
- **Startup key refresh on update**: Each Velopack delta update refreshes the startup registry entry so it always points to the current version.
- **Clean uninstall**: Uninstalling now kills the running tray and server and removes all startup entries, leaving nothing behind.

## Upgrade Notes

- **Windows**: The Velopack tray will apply this update automatically in the background. No action required.
- **Docker**: No changes to the container image. Pull the new tag and restart.
- No database or configuration migration is required.
